## Raw Feedback (Sanitized)

## Normalized Summary
The user wants a living `docs/design-guidelines.md` that formalizes the intended visual language — calm minimalism, warm neutrals, editorial typography, soft rounded components — so that implementers (human or AI) build new features in a coherent style.

## PM Notes
- User's attached brief is thorough and well-structured. Key design pillars from the brief:
  1. Style: Calm minimalism / AI-native productivity UI
  2. Layout: Center-stage interaction, peripheral navigation
  3. Color: Warm neutral palette (off-white/beige bg, one warm accent, dark-grey text, not pure black)
  4. Typography: Editorial headline + minimal UI sans-serif
  5. Components: Rounded, soft containers, minimal visual noise
  6. Interaction: AI workspace feel, not software dashboard
  7. Comparable: Claude, Notion AI, Linear (light), Arc, Apple editorial
- The existing token set in styles.css (warm parchment #F5F3EA, orange accent #D25722, Avenir Next) is already broadly aligned — the doc codifies and extends it.
- This is a foundation ticket: once the doc exists, all subsequent UI tickets reference it.
- Self-evolving angle: the design guidelines doc should also be readable by AI agents proposing UI changes, so it acts as a design guardrail for the self-evolution pipeline.

## Triage Decision
- Decision: create T-0066 (Design guidelines document), rank 1 in new epic
- Rationale: Foundation work that makes every subsequent UI ticket cheaper and more consistent. Docs-only → no QA run needed; a doc review suffices.
- Revisit Trigger: after T-0066 is accepted; guidelines should be updated whenever a new design decision is made
