## 3. Manual Scenario Review

### Scenario A — Happy path (apply + rollback)
- `pending_apply` artifact saved → `BackgroundTasks.add_task(apply_pipeline.apply, artifact)` fires after response
- `apply()` sets `applying` → `_check_base_ref` → `_sandboxed_validate` (npm) → `_apply_and_commit` (patch + git add + git commit) → `applied` with `git_commit_sha`
- Rollback: `POST /agent/rollback { patch_id }` → `pipeline.rollback()` → `git revert --no-edit` → `reverted` with `revert_commit_sha`
- **Result:** Correct end-to-end; verified by integration test `test_apply_then_rollback`

### Scenario B — Build-gate failure
- `_sandboxed_validate` applies diff to temp copy, runs `npm run validate`; on non-zero exit raises `ApplyError("validation_failed")`
- `apply()` catches → `apply_failed`, `failure_reason = "validation_failed"`, no commit
- Real working copy never touched (patch applied only to temp)
- **Result:** Correct; verified by `test_build_gate_failure_no_git_commit`

### Scenario C — base_ref mismatch
- `_check_base_ref` gets current HEAD; rejects if neither SHA is a prefix of the other
- Status → `apply_failed`, `failure_reason = "base_ref_mismatch"`
- **Result:** Correct; verified by `test_base_ref_mismatch_no_git_commit`

### Scenario D — Revert conflict
- `git revert --no-edit` exits non-zero → `git revert --abort` cleans up → `rollback_conflict`
- Working tree left clean (no partial merge state)
- **Result:** Correct; verified by `test_rollback_conflict_sets_status_and_raises`

### Scenario E — Patch not found (rollback)
- `PATCH /agent/rollback { patch_id: "not-exists" }` → 404 `patch_not_found`
- **Result:** Correct; verified by `test_rollback_nonexistent_patch_returns_404`

## 4. Edge-case / Risk Notes

| Risk | Assessment |
|---|---|
| Mid-apply inconsistency: patch applies to real files but git commit fails | `apply_failed` is set and stored; source files may be partially modified. Acceptable for M8 (out-of-scope recovery). No bug ticket needed — spec does not require atomic file rollback on git failure. |
| `git revert --abort` return code not checked | If `--abort` fails, working tree state is undefined. Low probability; `--abort` fails only when no revert is in progress. Risk is acceptable for M8. |
| `npm run validate` timeout (120s) | Hard-coded. If validate runs longer, apply will be stuck in `applying`. Acceptable for M8; no in-flight timer. |
| `_TEST_TMP_BASE` dir left in workspace after tests | `.test-tmp/` is created at module import. Should be added to `.gitignore`. |
