# QA Checkpoint — T-0062: Patch notification — dismiss affordance and human-readable failure reasons

- Ticket: T-0062
- Date: 2026-03-03
- Area: ui
- Scope: PatchNotification banner — applied + apply_failed states

## Test Plan

1. **Applied state — dismiss affordance**
   - Trigger a patch that reaches `status: applied` with a non-empty `unified_diff`.
   - Verify toast shows:
     - Primary action: **Undo** button.
     - Secondary actions: **See what changed** toggle (when diff present) and **Done** button.
   - Click **Done** and confirm:
     - Notification disappears.
     - Changelog entry in Settings still shows the patch with an Undo button.
2. **Apply failed — human-readable reasons**
   - For each `failure_reason` code, simulate `status: apply_failed` and verify copy:
     - `validation_failed` → contains "checks didn't pass".
     - `base_ref_mismatch` → contains "the codebase changed since this was generated — resubmit your feedback".
     - `empty_or_malformed_patch` → contains "the agent returned an unusable response".
     - `harness_unavailable` → contains "the AI agent couldn't be reached".
     - Unknown / `null` → contains "an unexpected error occurred".
   - In all cases, ensure the toast includes "Couldn't apply the change" and "No files were modified".
3. **Dismiss affordance — accessibility**
   - Ensure **Done** and **Dismiss** buttons:
     - Are rendered as `<button>` elements.
     - Are reachable via keyboard focus (tab order matches other actions).
     - Use the shared focus ring styles from `actionBtnBase`.
