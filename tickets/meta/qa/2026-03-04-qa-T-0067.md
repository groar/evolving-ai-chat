## UX/UI checklist (heuristic, Area: ui)

### 1) Mental model and framing — PASS
- Activity sheet title "Activity" and empty state copy make it clear this is history of AI-applied changes.
- Primary job (review history) is obvious; Undo is secondary in expanded card.

### 2) Hierarchy and focus — PASS
- Card list is primary; Undo is a secondary button inside expanded card.
- No "everything is a card" overload; date groups provide structure.

### 3) Information architecture — PASS
- Single entry: top-bar History icon (primary) and Settings "View activity →" (secondary). No duplicate changelog wall in Settings.

### 4) States and error handling — PASS
- Empty state: "No activity yet. Changes applied by AI will appear here." — clear, single message.
- Failed cards: red-orange border, failure reason on expand (per spec).

### 5) Copy and terminology — PASS
- "Activity" (not "Changelog"); "View activity →"; "No activity yet…". No promise control issues.

### 6) Affordances and interaction — PASS
- Cards are clickable to expand; Undo is a clear button. Sheet uses Radix (Escape to close, focus management).

### 7) Visual accessibility — PASS
- Uses existing design tokens (panel, border, muted). Heading "Activity", then date labels, then cards.

### 8) Responsive / window — PASS
- Sheet width `min(520px, 92vw)`; content scrolls. No clipped CTAs observed in implementation.

## UX acceptance criteria (ticket)

- Opening Activity with 10+ entries: layout is list + date groups; no crowding by design.
- No horizontal overflow: cards full-width, diff in `<pre>` with `max-h-80` and overflow-auto.
- Empty/error states: copy and styling per spec.
- Sheet keyboard-closeable (Escape): Radix Sheet provides this. Trigger is focusable (top-bar button).

## Copy regression
