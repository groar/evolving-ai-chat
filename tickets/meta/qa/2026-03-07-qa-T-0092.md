## Manual Scenario Analysis (Code Path Tracing)

**Normal flow (happy path):**
1. User submits feedback item "The greeting feels robotic" → `FeedbackPanel.onRequestCodePatch` fires
2. `App.tsx` closes improvement sheet, calls `refinement.start(feedbackId, feedbackTitle, feedbackSummary, "ui")`
3. `useRefinement.start()`: loads product context from `/agent/refine-context`, creates new conversation, sends feedback text as first user message with refinement system prompt
4. `App.tsx` conditionally renders `<RefinementConversation>` with "Refining feedback" badge + feedback title
5. User responds to clarifying question(s)
6. Model produces `**Goal**: ...` formatted description → action buttons appear below message
7. User clicks [Run Agent] → `requestPatch(feedbackId, feedbackTitle, feedbackSummary, "ui", { raw_description: lastAssistantText, refinement_conversation_id: convId })` called
8. Backend receives `refined_spec` → sets `feedback["refined_spec_text"]` → `_build_structured_prompt` uses it as primary content
9. Agent uses refined description instead of raw feedback summary ✓

**Edge cases verified:**
- Missing context docs: endpoint returns `docs_missing`, context may be empty, `useRefinement` continues with empty context (degraded mode) ✓
- Context fetch failure (offline): caught, proceeds with empty context, no error shown to user until message send fails ✓
- Auto-summarize at 10 user messages: nudge appended to user message text ✓
- [Cancel]: `refinement.cancel()` resets all state, main chat view restored ✓
- [Edit]: user can continue typing in composer for another refinement round ✓
- Patch in flight: `requestPatch` returns 409, `settingsError` set, user notified ✓
- `refined_spec` absent (original callers): `CodePatchRequest.refined_spec = None` → existing behavior unchanged ✓

## UX/UI Design QA (Area: ui)

**1. Mental Model and Framing** — PASS
- "Refining feedback" badge (pulsing orange dot) clearly indicates active mode
- Truncated feedback title in badge header ties the conversation to its origin
- "Cancel" always visible in mode bar; no user can feel trapped
- `<h1>` at top still shows conversation/app name for orientation

**2. Hierarchy and Focus** — PASS
- [Run Agent] is primary (filled orange button), [Edit] and [Cancel] are secondary (white)
- Action buttons only appear when the model produces a functional description (conditional rendering) — no premature button confusion
- Composer placeholder changes context-appropriately when buttons are visible

**3. Information Architecture** — PASS
- Refinement replaces the main chat area entirely (not a modal/sheet) — user knows they're in a distinct mode
- Improvement sheet closes when refinement opens — no dual modal confusion
- Cancel in the refinement header bar (top) provides a stable escape hatch

**4. States and Error Handling** — PASS
- Loading: "Setting up refinement conversation…" shown while fetching context and creating conversation
- Error: red `role="alert"` banner when error is set (connection failure, etc.)
- Streaming: blink cursor during assistant response (matches main chat behavior)
- WARN (acceptable): Degraded context mode (missing docs) is silent to the user — by spec design, this is correct behavior. Risk is low: the model still functions without context.

**5. Copy and Terminology** — PASS
- "Refining feedback" — clear, functional, not technical
- "Product analyst" — sets appropriate expectation for the assistant role
- "Ready to proceed with this description?" — clear intent before action buttons
- [Run Agent] [Edit] [Cancel] — consistent with spec §4.2 and user-facing AC
- "Ask a follow-up or provide more detail…" — helpful placeholder
- Copy regression: no existing copy changed. New copy does not imply unavailable behaviors.

**6. Affordances** — PASS
- Enter-to-send works in refinement composer (same as main chat)
- Send button disabled when loading or empty input
- [Run Agent] clearly primary by styling
- Focus management: `setTimeout(() => inputRef.current?.focus(), 50)` after sending

**7. Visual Accessibility** — PASS
- Same color token system as main chat (`border-border`, `#efbe91`, etc.)
- Pulsing dot uses animation only for the status indicator (not distracting)
- `role="alert"` on error state for screen reader compatibility
- `aria-live="polite"` on messages container

**8. Responsive** — PASS
- `max-w-[700px]` on messages (same as main chat)
- Header badge truncates feedback title with `max-w-[200px]` and `title` attr for full text

**Checklist result: PASS** — no bug tickets required
