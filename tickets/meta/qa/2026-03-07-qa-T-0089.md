## Manual Scenarios

**Normal flow — prompt assembly with mocked subprocess**:
Confirmed via `PromptAssemblyTests.test_all_required_sections_present` and `test_allowlist_patterns_in_scope_constraints`. The 5 patterns from `patch-allowlist.json` (`^apps/desktop/src/`, `^apps/desktop/runtime/`, `^tickets/`, `^docs/`, `^tests/`) all appear in the SCOPE CONSTRAINTS section.

**Edge case — retry context injection**:
`test_previous_attempt_section_present_when_retry_context_given` confirms that passing a non-None `retry_context` string appends a `=== PREVIOUS ATTEMPT ===` section containing the failure details. T-0091 can inject failure context into the same template without any further prompt changes (as specified).

**Edge case — git unavailable**:
Code review confirms that both `subprocess.run(['git', 'ls-files', ...])` and `subprocess.run(['git', 'log', '--oneline', '-5'])` are wrapped in `try/except (TimeoutExpired, FileNotFoundError)` with fallback strings `"(file tree unavailable)"` and `"(git log unavailable)"`. The prompt assembles successfully in degraded mode.

**Edge case — very large file tree (budget truncation)**:
`test_context_total_chars_within_budget` confirms total assembled context stays ≤ 32,200 chars (~8,000 tokens) even with 10,000 synthetic file paths. `recent_commits` is capped at 2,000 chars; the file tree fills the remaining budget and is truncated with a `... (truncated)` suffix.
