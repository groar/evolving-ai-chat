## Notes

**`pending_apply` vs `applying` in POST response**: The spec §4 shows the response status as `"applying"`, but the endpoint returns `"pending_apply"`. This is correct for T-0059 scope: the apply loop (git commit + build gate) is T-0060, which will transition the artifact from `pending_apply` → `applying` → `applied/apply_failed`. No defect.

**pi integration model**: pi.dev is a local CLI agent, not a remote API. The harness copies `apps/desktop/` to a temp sandbox, runs `pi -p "..." --no-session --tools read,write,edit,grep,find,ls --append-system-prompt "<scope guard>"`, then diffs `apps/desktop/src/` before vs after. This is the correct integration approach and is captured in the ticket and `.env.example`.
