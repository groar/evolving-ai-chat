## 3. UX/UI Design QA Checklist (Area: ui)

### 1. Mental Model and Framing — PASS
- Within 10 seconds a user sees: "Settings" title (SheetTitle), API key entry, Release Channel toggle.
- Primary job-to-be-done (configure the app) is the immediate surface.

### 2. Hierarchy and Focus — PASS
- Connections is structurally first; no competing peer sections above the fold.
- Activity is read-only (no editable action); clearly secondary.
- Advanced and Danger Zone are collapsed; visually absent until invoked.

### 3. Information Architecture and Navigation — PASS
- Advanced (beta-only) behind collapse → progressive disclosure ✓
- Danger Zone behind collapse → progressive disclosure ✓
- No duplicated control surfaces: Danger Zone removed from App.tsx, now only in SettingsPanel ✓.
- Single "View activity →" link to Activity sheet ✓.

### 4. States and Error Handling — PASS
- Notices/errors above Connections: time-sensitive messages appear first ✓.
- API key error renders inline within Connections (scoped to the input) ✓.
- No repeated banners: global `error` and `apiKeyError` serve different scopes ✓.

### 5. Copy and Terminology — PASS with WARN
- "Release Channel" replaces "Updates" — clearer, maps to mental model ✓.
- "Advanced (beta-only)" — communicates it's not for standard users ✓.
- "Danger Zone" — explicit but collapsed/not alarming for new users ✓.
- "This cannot be undone" — appropriate destructive warning ✓.
- No duplicated instructions ✓.
- WARN: Section labels use `text-[10px]` (10 px) instead of `text-xs` (12 px) as specified in the ticket. At 10 px, uppercase letter-spaced labels remain visually legible (uppercase characters appear ~20% larger than their point size), and this matches common patterns for section separators. **Judgment: acceptable to ship.** No follow-up ticket created; note recorded here. If user feedback flags readability, bump to `text-xs` in a micro-ticket.

### 6. Affordances and Interaction — PASS
- `<details>/<summary>` is keyboard-navigable (native HTML, Enter/Space to toggle) ✓.
- Delete Local Data button disabled while `isBusy` ✓.
- "This cannot be undone" copy sets expectation before any click ✓.

### 7. Visual Accessibility Basics — PASS
- Section label contrast: `text-muted-foreground` (secondary text) is used consistently ✓.
- Main body text remains `text-foreground` ✓.
- Spacing: `gap-3` within sections, `pt-5` between sections, `mt-1` before details — consistent vertical rhythm ✓.

### 8. Responsive / Window Size — PASS
- Sheet width: `w-[min(400px,90vw)]` with `overflow-y-auto` (from App.tsx) ✓.
- Danger Zone and Advanced are at the bottom and collapsed; no content clipped on narrow viewports ✓.
